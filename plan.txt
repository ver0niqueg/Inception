Ce projet consiste a mettre en place une mini-infrastructure de differents services en suivant des regles specifiques. → il faut relier plusieurs image Docker et pouvoir les lancer ensemble sans pour autant qu’elles perdent leur independance (grace a Docker Compose).

Docker sert a créer, executer et gerer des applications dans des ‘conteneurs’.

1. Le problème que Docker résout
Avant Docker, installer une application sur une machine impliquait :
    • installer la bonne version de Python, Java, Node, etc.
    • configurer les dépendances et bibliothèques
    • vérifier que tout fonctionne pareil sur toutes les machines
👉 Résultat : “Chez moi ça marche, mais pas chez toi.”

2. Ce que fait Docker
Docker crée un conteneur, c’est-à-dire un petit environnement isolé avec :
    • ton application
    • ses dépendances
    • son système minimal (bibliothèques, variables, ports, etc.)
Tu peux alors :
    • lancer la même appli partout (Mac, Windows, Linux, serveur, cloud)
    • éviter les conflits entre projets
    • déployer facilement sur plusieurs machines


Docker est un outil qui peut empaqueter une application et ses dependances dans un conteneur isole.
L’avantage de Docker est la possibilite de modeliser chaque conteneur sous la forme d’une image que l’on peut stocker localement.
Un conteneur est une machine virtuelle sans noyau → il ne contient que l’application et les dependances de l’application.

Un Dockerfile est le nom du fichier principal des images Docker.

Un conteneur Docker contient tout ce dont l’application a besoin pour tourner : le code, les bibliothèques, les dépendances et parfois même certaines configurations système.
Comme tout est emballé dans ce conteneur, il fonctionne de la même façon sur n’importe quelle machine qui a Docker installé, que ce soit ton PC, un serveur Linux, ou le cloud.
C’est un peu comme un “mini-ordinateur virtuel” léger, mais contrairement à une machine virtuelle classique, il partage le noyau du système d’exploitation avec la machine hôte, donc il consomme beaucoup moins de ressources et démarre très rapidement.
Docker Compose est un outil qui a ete developpe pour aider a definir et a partager des applications multi-conteneurs. Avec Compose, nous pouvons créer un fichier YAML pour definir les services et a laide d’une seule commande, tout mettre en route ou tout demonter.
Compose permettrait donc de gérer des applications qui utilisent plusieurs containers et de communiquer entre eux.

# ✅ **Étape 3 — Construire chaque service Docker un par un**

Tu vas créer **3 images** à partir de 3 Dockerfiles :

### **1) MariaDB**

* Dockerfile
* script d’initialisation SQL
* configuration (my.cnf)
* volume persistant `/var/lib/mysql`

### **2) WordPress + PHP-FPM**

* Dockerfile (tu dois installer WordPress via `wp-cli`)
* connexion à MariaDB via variables d’environnement
* volume persistant `/var/www/wordpress`

### **3) Nginx**

* Dockerfile
* config Nginx (TLS, fastcgi, etc.)
* certificat SSL auto-signé
* il sert WordPress via PHP-FPM

➡️ Le tout doit être orchestré par `docker-compose.yml`.

---

# ✅ **Étape 6 — Configurer le Makefile**

Ton Makefile doit contenir **au minimum** :

```
all: up
up:
    docker compose -f srcs/docker-compose.yml up --build -d
down:
    docker compose -f srcs/docker-compose.yml down
clean:
    docker system prune -af
fclean: down
    docker volume rm -f $(docker volume ls -q)
re: fclean all
```

➡️ L’évaluateur testera **exclusivement** avec `make`.

What is Docker 🐳 ?

Docker est une plateforme open-source qui permet de créer, déployer et exécuter des applications dans des conteneurs. Un conteneur est une unité légère, portable et isolée qui embarque tout le nécessaire pour faire fonctionner une application (code, librairies, dépendances, configuration…).

Contrairement aux machines virtuelles, les conteneurs partagent le noyau du système hôte, ce qui les rend plus rapides, plus légers et plus efficaces. Docker permet ainsi de garantir que l'application se comporte de la même façon partout, quel que soit l'environnement (développement, test, production...).
Multi-container orchestration
docker-compose.yml 	fichier central du projet
permet 	de mettre en relation plusieurs conteneurs Docker, il permet donc de créer un réseau Docker
définit et configure 	les services (conteneurs) à exécuter ensemble, leurs volumes, réseaux, et les variables d’environnement nécessaires
Persistant storage

volumes:    # espaces de stockage partagés entre conteneurs pour persister les données au-delà du cycle de vie des conteneurs
  mariadb_data:
    driver: local
    driver_opts:
      type: none    # pas de système de fichiers spécial, utilisation du système hôte
      device: /home/${USER}/data/mariadb   # dossier sur la machine hôte lié au conteneur
      o: bind       # bind mount : partage direct du dossier hôte dans le conteneur
